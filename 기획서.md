# TRADE CHASER — 기획서

> **"밤사이 미장, 아침에 3분 브리핑"**
> 내가 찾아보지 않아도 → 밤사이 미국 시장 흐름을 요약해주고 → 내 관심종목이 어떤 영향을 받았는지 정리하고 → 오늘 무엇을 확인하면 되는지까지 이어주는 **AI 미장 브리핑 서비스**

- 몰입캠프 2주차 공통과제 II — **LLM Wrapper** 옵션
- 2인 1팀 / 개발 기간 1주
- 문서 버전: v1.0 (2026-07-10)

---

## 1. 배경 & 문제의식

미국 주식(미장)은 **한국 시간 밤 23:30 ~ 새벽 06:00**에 열린다. 국내 투자자 입장에서:

- 장이 열리는 동안은 **자고 있어서 볼 수가 없다.**
- 아침에 일어나면 이미 상황은 다 끝나 있고, 무슨 일이 있었는지 파악하려면 **여러 뉴스·앱을 직접 뒤져야 한다.**
- 정보는 넘치지만, **"그래서 내 종목에 무슨 의미인지"** 를 해석해주는 서비스는 드물다.

→ **핵심 인사이트:** 미장은 실시간으로 볼 수 없는 게 자연스럽다. 그렇다면 실시간 대신 **"자는 동안 벌어진 일을 아침에 정리해서 떠먹여 주는"** 모델이 가장 합리적이다. 시간대 차이가 약점이 아니라 **제품 컨셉의 근거**가 된다.

## 2. 타겟 사용자

| 페르소나 | 특징 | 니즈 |
| --- | --- | --- |
| **바쁜 직장인 투자자** | 미국 대형주 몇 종목 보유, 장을 실시간으로 볼 시간 없음 | 아침에 핵심만 빠르게 |
| **미장 입문자** | 관심은 있지만 영어 뉴스·용어가 벽 | 한국어로 쉽게 해석된 브리핑 |
| **정보 과부하 투자자** | 뉴스는 많이 보는데 정리가 안 됨 | "사실 vs 해석"이 분리된 요약 |

공통점: **직접 찾아보는 수고 없이, 필요한 정보만 습득하고 싶다.**

## 3. 핵심 가치 & 차별점

### 3-1. 한 줄 가치 제안
> "미장 뉴스를 다 읽을 시간은 없지만, 내 종목이 밤사이 어떻게 됐는지는 알고 싶은 사람"을 위한 **AI 아침 브리핑.**

### 3-2. 경쟁 서비스 대비 차별점

| 서비스 | 비슷한 점 | 우리가 다르게 하는 점 |
| --- | --- | --- |
| 토스증권 AI Signal | AI로 시장 움직임 분석 | **밤사이→아침 브리핑**이라는 시점 특화, 개인 관심종목 기준 해석 |
| Stock Briefing | 보유 종목 뉴스·감성 요약 | **사실/해석 명확히 분리**, 원문 인용 강제 |
| AInvest | AI 뉴스·종목 추천·트래커 | 매매 추천이 아니라 **정보·관점·"오늘 확인할 것"** 중심 |
| Stock-Alert.AI | 뉴스 감성·펀더멘털 알림 | 알림보다 **"오늘 내 종목에 미친 영향도"** 서술 중심 |
| 일반 증권앱 | 뉴스·차트·관심종목 | 사용자 **투자 유형별 맞춤 해석 + 영어 뉴스 한국어화** |

### 3-3. 방어 가능한 3가지 축
1. **시점 특화** — "밤사이 → 아침" 브리핑이라는 명확한 사용 맥락.
2. **해석 중심** — 매매 지시(규제·신뢰 리스크)가 아니라 "정보 습득 → 관점 → 확인할 액션".
3. **한국어화** — 영어 미장 뉴스를 Claude가 한국어로 요약·긍부정 분리.

## 4. 기능 정의

### 4-1. 필수 기능 (MVP)
- **회원가입 / 로그인** (앱 자체 계정 — 증권 계좌 불필요)
- **관심 종목 등록** (미국 티커: AAPL, NVDA, TSLA …)
- **관심 종목 섹터 분류** (정적 매핑 테이블 기반)
- **밤사이 뉴스·시황 수집** (Finnhub / 뉴스 API → 종목별)

### 4-2. 핵심 기능 (LLM Wrapper 본체)
아침 브리핑 = 아래 **4단 구조**로 생성:

```
① 전체 시황 요약    지수·섹터의 큰 흐름 (밤사이 무슨 일이)
② 내 종목별 영향    관심종목마다 긍정 / 부정 / 주의 이슈 정리
③ 근거(사실)        각 판단의 출처 = 원문 뉴스 인용 (환각 방지 + 신뢰)
④ 오늘 확인할 것    "매매 지시"가 아닌 정보·관점·액션 제안
```

②③은 아래 표 구조로 저장·렌더링:

| 이슈 | 관련 종목 | 영향 | 이유 | 출처 |
| --- | --- | --- | --- | --- |
| 미 연준 금리 인하 시사 | NVDA | 긍정 | 성장주 밸류에이션 부담 완화 | (뉴스 링크) |
| 달러 강세 지속 | AAPL | 중립~부정 | 해외 매출 환산 부담 | (뉴스 링크) |

### 4-4. 커스터마이즈 분석 렌즈 (신규 핵심)
사용자가 **리포트·증권 기사·다른 LLM 리포트**를 넣고, **분석 카테고리 + 성향(톤)** 을 직접 골라 분석시키는 기능. LLM Wrapper의 "wrapper"를 제품화하는 차별점.

- **입력:** 뉴스 API / 텍스트 붙여넣기 (여력 시 PDF 리포트 업로드)
- **카테고리 선택:** 지수·지표·섹터·테마 (예: SOX, US10Y, 반도체) — [분석카테고리.md](분석카테고리.md)
- **성향 선택:** 거시분석 / 가치투자 / 모멘텀 / 팩트 / 입문자용 등 프리셋
- **동작:** 2단계 파이프라인(팩트 추출 → 성향 렌더링) — 7-1 참고
- **데모 포인트:** "같은 뉴스를 성향만 바꿔 두 버전 비교"

### 4-3. 선택 기능 (여력 시)
- **관심종목 랭킹** — 다른 사용자들이 많이 등록한 종목 Top N (다중 사용자 요소 강화)
- **종목 코멘트** — 같은 종목을 보는 사용자들의 짧은 의견 공유
- **투자 유형별 브리핑** — 안정형 / 공격형에 따라 해석 톤 조정
- **모바일 반응형 UI**
- **관심도 변화 반영** — 등록 수 급증 종목 하이라이트

## 6. LLM 설계 (핵심)

### 6-1. 파이프라인 (2단계 — 커스터마이즈 렌즈 지원)
```
[입력]  뉴스/리포트/붙여넣기 텍스트
   │
   ▼  1단계 · 팩트 추출 (중립·사실만)  ─────▶ facts JSON ──(문서·날짜로 캐싱, 공유)
   │
   ▼  2단계 · 성향 렌더링 (사용자 렌즈: 카테고리+성향+심층도)
   │        └ 팩트 JSON만 재서술 → 저렴, 캐시 유지
   ▼
[구조화 JSON] ─▶ DB 캐싱 ─▶ 프론트 렌더
```
- **1단계**는 무거운 원문 읽기를 한 번만 하고 캐싱 → 비용↓, 환각↓(사실 고정).
- **2단계**만 사용자별로 변형 → 맞춤성 + 저비용 동시 확보.
- 상세 문안: [프롬프트템플릿.md](프롬프트템플릿.md)

### 6-2. 출력 스키마 (Claude가 반환할 JSON)
```json
{
  "market_summary": "지수·섹터 큰 흐름 3~4문장",
  "sentiment": "positive | neutral | negative",
  "positive_factors": ["...", "..."],
  "negative_factors": ["...", "..."],
  "watch_issues": ["오늘 주의할 이슈"],
  "reasons": [
    { "factor": "금리 인하 시사", "impact": "긍정",
      "explain": "...", "source_url": "https://..." }
  ],
  "today_actions": ["매매 지시 아님 — 확인할 것 / 관점"]
}
```

### 6-3. 환각 방지 (금융 도메인 최대 리스크)
- **"입력으로 제공된 뉴스 텍스트 안에서만 근거를 찾아라"** 를 시스템 프롬프트로 강제.
- 모든 `reasons` 항목에 **원문 `source_url` 필드 필수** → 근거 없는 서술 차단.
- 수치·실적은 **뉴스에 명시된 것만** 인용, 추정 금지.
- Claude **prompt caching** 으로 공통 시스템 프롬프트·종목 컨텍스트 토큰 절감.

### 6-4. "추천"의 안전한 프레이밍
- ❌ 금지: "지금 NVDA 사세요" (투자자문 규제·신뢰 리스크)
- ✅ 지향: "오늘 이 이슈를 확인해보세요 / 이런 관점에서 볼 수 있습니다 / 함께 볼 섹터"
- 화면 하단 고지: *"본 브리핑은 정보 제공 목적이며 투자 권유가 아닙니다."*

## 7. 시스템 아키텍처

### 7-1. 실시간 대신 "1일 1회 배치 + 캐싱"
미장 마감(한국 아침 06시경) 직후 **하루 한 번** 종목별 브리핑을 생성해 저장. 사용자는 그 캐시를 조회.

```
[스케줄러]  매일 07:00 KST 트리거
     │
     ▼
[브리핑 생성 배치]
   1. 활성 관심종목 목록 수집
   2. 종목별 밤사이 뉴스 + 종가 조회
   3. Claude 호출 → 구조화 브리핑
   4. daily_briefings 테이블에 저장
     │
     ▼
[사용자 조회]  아침에 앱 열기 → 캐시된 브리핑 반환 (LLM 재호출 없음)
```

- **비용:** LLM 호출 = "브리핑 대상 종목 수 × 1회/일" → 사용자 수와 무관하게 저렴.
- **인프라:** 상시 WebSocket 불필요 → 단순 REST + 크론/스케줄러 하나.
- **on-demand 보완:** 캐시에 없는 신규 종목은 최초 조회 시 생성 후 캐싱.

### 7-2. 데이터 소싱 & 비용/라이선스
| 데이터 | 소스 | 비용 | 라이선스 |
| --- | --- | --- | --- |
| 미국 종목 시세(종가·등락) | Finnhub / yfinance | 무료 | 지연·종가라 재배포 안전 |
| 종목 뉴스 | Finnhub Company News | 무료 티어 | 안전 (요약·인용) |
| LLM 해석 | Claude API | 유료(캐싱으로 절감) | — |

> 실시간 거래소 시세를 공개 배포하지 않으므로 라이선스 이슈 없음. 화면에 *"지연/종가 기준"* 표기.
