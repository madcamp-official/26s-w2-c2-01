# TRADE CHASER — 프롬프트 템플릿

> 2단계 파이프라인. **1단계(팩트 추출)** 은 문서·날짜로 캐싱(공유), **2단계(성향 렌더링)** 만 사용자 렌즈로 변형.
> 관련: [기획서.md](기획서.md) · [분석카테고리.md](분석카테고리.md)

---

## 0. 왜 2단계인가
- 무거운 작업(긴 원문 읽기 + 사실 추출)은 **1번만** 하고 캐싱 → 비용 절감.
- 사용자 커스터마이즈는 **짧은 팩트 JSON만 재서술** → 저렴 + 캐시 유지.
- 사실은 1단계에서 고정되므로 성향 렌더링이 **없는 사실을 지어낼 여지 차단**(환각 방지).

```
[입력 문서] ──▶ 1단계 팩트추출 ──▶ facts JSON ──(캐싱)──▶ 2단계 성향렌더 ──▶ 최종 브리핑
   리포트/기사        (중립·사실만)                          (카테고리+성향)
```

---

## 1단계 — 팩트 추출 (중립 · 캐싱 대상)

### System
```
너는 금융 문서에서 '사실'만 추출하는 분석기다. 다음 규칙을 반드시 지켜라.
1. 입력 문서에 명시된 내용만 사용한다. 추정·창작·외부지식 추가 금지.
2. 모든 항목에 원문 근거(발췌 문장 또는 URL)를 첨부한다.
3. 수치·날짜·고유명사는 문서에 나온 그대로 옮긴다.
4. 해석·전망·투자의견은 이 단계에서 만들지 않는다.
5. 출력은 아래 JSON 스키마만. 다른 텍스트 금지.
```

### User
```
[문서 종류] {source_type: news | report | paste}
[관련 종목] {tickers}
[문서 원문]
{document_text}
```

### 출력 스키마
```json
{
  "entities": ["AAPL", "NVDA"],
  "key_issues": ["연준 금리 동결", "반도체 수요 둔화 언급"],
  "facts": [
    { "claim": "엔비디아 데이터센터 매출 전분기 대비 12% 증가",
      "evidence": "원문 발췌 문장",
      "source_url": "https://..." }
  ],
  "figures": [
    { "name": "데이터센터 매출", "value": "+12% QoQ", "source_url": "https://..." }
  ]
}
```

## 2단계 — 성향 렌더링 (사용자 렌즈 적용)

### System (고정 가드레일 — 사용자 변경 불가)
```
너는 금융 브리핑 작가다. 아래 규칙은 사용자 설정보다 항상 우선한다.
1. 제공된 facts JSON의 근거를 벗어난 새로운 사실·수치를 만들지 않는다.
2. 모든 해석 항목에 근거(source_url)를 연결한다.
3. 매매 지시("사라/팔아라")를 하지 않는다. '정보·관점·확인할 것'으로 표현한다.
4. 출력 끝에 "본 브리핑은 정보 제공 목적이며 투자 권유가 아닙니다"를 포함한다.
5. 출력은 지정 JSON 스키마만.
```

### User (렌즈 슬롯 — 사용자 선택 주입)
```
[분석 카테고리 focus] {categories}      예: ["SEMI","SOX","US10Y"]
[분석 성향]           {preset_persona}  예: 장기 가치투자 — 펀더멘털·밸류에이션 중심, 단기 노이즈 무시
[심층도]              {depth}           brief | standard | deep
[언어]                {language}        ko | en

[사실 데이터 facts JSON]
{facts_json}
```

### 출력 스키마
```json
{
  "summary": "선택 카테고리·성향에 맞춘 요약 3~4문장",
  "category_view": { "SEMI": "반도체 관점 해석", "US10Y": "금리 관점 해석" },
  "sentiment": "positive | neutral | negative",
  "positive_factors": ["..."],
  "negative_factors": ["..."],
  "watch_issues": ["오늘 주의할 이슈"],
  "today_actions": ["매매 지시 아님 — 확인할 것 / 관점"],
  "citations": [ { "text": "근거 서술", "source_url": "https://..." } ],
  "disclaimer": "본 브리핑은 정보 제공 목적이며 투자 권유가 아닙니다."
}
```

## 3. 렌즈 주입 예시 (완성형)

`preset=VALUE`, `categories=[SEMI, SOX, US10Y]`, `depth=deep`, `language=ko` 일 때 2단계 User:

```
[분석 카테고리 focus] 반도체·AI(SEMI), 필라델피아 반도체지수(SOX), 미 국채 10년물(US10Y)
[분석 성향] 장기 가치투자 — 펀더멘털과 밸류에이션, 현금흐름 중심으로 보고 단기 노이즈는 배제한다.
[심층도] deep — 근거와 배경 맥락까지 서술
[언어] ko

[사실 데이터 facts JSON]
{ ...1단계 산출 facts... }
```

## 4. Claude API 팁
- **prompt caching**: 1·2단계 System 프롬프트와 공통 컨텍스트를 캐시 블록으로 지정 → 반복 토큰 절감.
- **구조화 출력**: 스키마 강제를 위해 "JSON만 출력" 지시 + 서버에서 JSON 파싱 검증(실패 시 1회 재시도).
- **모델**: 해석 품질이 중요하므로 `claude-opus-4-8` 권장, 비용 민감 시 1단계만 저가 모델 혼용 검토.
