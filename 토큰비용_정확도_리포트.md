> ⚠️ **정확도는 측정값이 아닌 정성 추정치입니다.**
> 아래 정확도(%)는 모델 지능 등급과 입력 정보량을 근거로 한 상대 추정이며, 벤치마크 수치가 아닙니다. 실제 값은 정답 10~20건으로 소규모 eval을 돌려야 확정됩니다. 토큰·비용은 산식 기반 실측 추정입니다.

## 3. 예상 정확도 (정성 추정)
두 축의 곱: **입력 정보량**(제목 < 스니펫 < 원문)이 상한을 정하고, **모델 해석력**(Haiku < Sonnet < Opus)이 그 안에서 실현도를 정합니다.

| 입력 방식 | Haiku 4.5 | Sonnet 5 | Opus 4.8 |
|---|---|---|---|
| 제목만 | ~60% | ~66% | ~70% |
| 제목+스니펫 | ~76% | **~86%** | **~90%** |
| 원문 전체 | ~81% | ~90% | ~95% |
*정확도 = "핵심 이슈를 빠짐없이, 근거 있게, 환각 없이" 잡는 정도의 상대 추정. 제목만은 모델이 빈틈을 상상으로 메워 환각 위험이 높음.*

## 4. 비용 × 정확도 요약 (기사 10건 기준)

SVG 그래프의 핵심 시사점은 다음과 같습니다:
* **가성비 스위트스팟:** **스니펫+Opus(~90%, $0.038)** ≈ **원문+Sonnet(~90%, $0.035)**
* 원문+Opus는 정확도 +5%p를 위해 2배 이상의 비용을 지불해야 하므로 완벽한 수확 체감 구간에 해당합니다.

## 5. 권장 구성 & 실제 운영비

### ★ 추천: 제목+스니펫 10건 · 2단계 모델 믹스
무거운 **① 팩트 추출은 Sonnet 5**, 품질이 중요한 **② 성향 렌더링은 Opus 4.8**로 분리합니다.

| 단계 | 모델 | 입력/출력 토큰 | 비용 |
|---|---|---|---|
| ① 팩트 추출 | Sonnet 5 | ~2,600 / 600 | $0.011 |
| ② 성향 렌더 | Opus 4.8 | ~1,000 / 700 | $0.023 |
| **합계 / 종목 / 일** | | | **≈ $0.034** |

**캠프 규모(고유 50종목 · 1주일 운영 시):**
* 50 × $0.034 × 7 ≈ **$12** (캐싱으로 사용자 수와 무관하게 고정 유지).

## 6. 결론

* **원문 전체는 비권장** — 정확도 +5%p 얻자고 비용 2~4배 증가 및 크롤링/파싱 리스크 발생. 전형적인 수확 체감 구간.
* **제목만도 비권장** — 저렴하지만 정보 부족으로 환각 위험↑, 정확도 상한 ~70%.
* **스위트스팟 = 제목+스니펫 10건** — 정보량 대비 토큰 효율이 가장 최적화된 구간.
* **모델은 2단계 믹스** — 대량 읽기(①)는 Sonnet/Haiku, 해석(②)은 Opus를 사용하여 비용은 낮추고 품질은 유지.
* **비용은 "넣는 데이터 양"에 정비례** — 크롤링 양이 아니라 프롬프트 투입량이 변수. 필터·요약이 핵심 레버가 됨.
