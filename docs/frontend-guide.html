01 큰 그림

코드를 열기 전에 알아둘 3가지.

  • SPA(단일 페이지 앱) — 페이지 이동이 없습니다. App.jsx가 view 상태 하나로 어떤 화면을 그릴지 정합니다. (라우터 라이브러리 없음)
  • 전부 목업 — 종목·브리핑·추천이 src/data.js에 하드코딩돼 있습니다. 서버 호출(fetch)이 아직 한 줄도 없습니다.
  • 상태는 위에서 아래로 — 거의 모든 상태가 App.jsx에 useState로 있고, 자식 컴포넌트엔 props로 내려줍니다(prop drilling). 상태관리 라이브러리(Redux 등) 없음.
그래서 다음 할 일은 UI는 이미 다 만들어져 있으니, §09처럼 data.js 목업을 백엔드 API 호출로 하나씩 바꾸면 실제로 동작하는 앱이 됩니다.

03 화면 구조 · 네비게이션

라우터 대신 view 상태 하나로 화면을 바꿉니다.

briefing
오늘의 브리핑
⇄
briefing-detail
종목/섹터/전체 상세
⇄
mypage
프로필·관심종목
⇄
lens
분석 렌즈 편집

App.jsx는 view 값에 따라 해당 페이지 컴포넌트 하나를 조건부 렌더링합니다. 사이드바 클릭이나 카드 클릭이 setView(...)를 호출해 화면을 전환합니다.

const [view, setView] = useState('briefing');
// ...
{view === 'briefing' && <BriefingPage ... />}
{view === 'mypage'   && <MyPage ... />}
{view === 'lens'     && <LensPage ... />}

headerInfo()가 현재 view에 맞는 breadcrumb·제목을 계산해 TopBar에 넘겨줍니다.

04 상태 관리 (App.jsx)

중요한 상태가 전부 여기 모여 있습니다. 이걸 이해하면 절반은 끝.

상태의미
view지금 보여줄 화면 (briefing / detail / mypage / lens)
watch내 관심종목 티커 배열 ['NVDA','TSLA','AAPL']
lensByTicker종목별 분석 렌즈 설정 맵 {ticker → {cats, preset, depth, note}}
period / briefingTab브리핑 기간(밤사이/3일/주) · 탭(내종목/섹터/전체)
detail상세 화면 대상 {type:'stock', ticker} 등
profile닉네임·이메일·투자성향 (지금은 하드코딩)
theme / systemDark라이트/다크 (§08)

상태를 바꾸는 함수들(handleAddStock, removeFromWatch, openLens, updateLensFor 등)도 App.jsx에 있고, 자식에게 props로 내려갑니다. 예를 들어 검색해서 종목을 추가하면 watch에 넣고 곧바로 그 종목의 렌즈 화면으로 이동합니다.

const handleAddStock = useCallback((t) => {
  setWatch((prev) => prev.includes(t) ? prev : [...prev, t]);
  setStockSearch('');
  openLens(t);              // 추가하면 바로 렌즈 편집으로
}, [openLens]);

recFor(ticker)가 종목 섹터에 맞는 추천 렌즈(기본/대안)와 그 이유를 제공합니다. "추천" 버튼을 누르면 그 조합이 한 번에 적용되고, "왜 이 추천인가요?" 설명이 뜹니다. 하단의 "근거 인용 필수 · 매매 지시 금지"는 성향과 무관하게 고정된 가드레일로 표시됩니다.

백엔드와의 연결 여기서 조립한 렌즈(카테고리·성향·심층도)가 나중에 백엔드 /analysis 파이프라인의 2단계 입력으로 전달됩니다. 지금은 미리보기까지만, 저장·전송은 연동 예정.

08 테마 시스템

CSS 변수 + data-theme 속성. 백엔드 공부 자료와 같은 원리.

  • index.css가 :root에 색 변수(--accent, --t1, --card…)를 정의합니다.
  • 기본은 OS 설정(prefers-color-scheme)을 따르고, 사용자가 토글하면 <html data-theme="dark">가 붙어 그게 우선합니다.
  • App.jsx의 effectiveTheme이 실제 적용 테마를 계산하고, useEffect가 documentElement에 속성을 씁니다.
const effectiveTheme = theme || (systemDark ? 'dark' : 'light');
useEffect(() => {
  if (theme) document.documentElement.setAttribute('data-theme', theme);
  else document.documentElement.removeAttribute('data-theme');
}, [theme]);

09 백엔드 연동 지점

지금 (목업)바꿀 API위치

STOCKS 검색GET /stocks?search=BriefingPage

연동 팁 토큰 인증이 필요하므로, 로그인 후 받은 access_token을 저장했다가 요청 헤더에 Authorization: Bearer …로 넣어야 합니다. 백엔드는 이미 localhost:5173을 CORS 허용해 뒀습니다. fetch 로직은 src/api.js 같은 파일로 모아두면 깔끔합니다.
